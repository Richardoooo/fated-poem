# 整体任务栏
整体界面依然是一个垂直布局的【主容器】(Main Container)，内部从上到下按逻辑分为了三个主要的【独立卡片区块】(Card / Panel)，每个区块都带有轻微的背景色和圆角边框，依次垂直排列。

一、 态势概览区块 (Overview Panel)
- 【头部标题容器】：位于区块顶部，包含左侧的一个装饰性图标以及紧跟其后的主标题文本（如“任务概览”类提示）。
- 【统计数据网格】(Grid/Flex Row Container)：包含两个水平排列的子模块，左右等分整个区域宽度，并带有边框包裹。
  - 左侧统计模块：上方为分类标签（如全部任务），下方为大号数字文本。
  - 右侧统计模块：上方为分类标签（如某个特定状态的任务），下方为大号数字文本。
- 【操作焦点区域】(Focus Section)：
  - 【前置提示文本】：两行上下排列的小号辅助说明文本。
  - 【下拉选择控件】(Dropdown / Select Box)：一个占据整行的交互控件，内部左侧为占位提示文本，最右侧是一个下拉箭头图标。
  - 【底部辅助文本】：位于下拉框下方的一段提示性说明文本。

二、 导航/筛选栏区块 (Filter/Tab Bar)
- 【水平菜单容器】(Flex Row Container)：一个较窄的横向卡片，内部采用水平弹性布局。
  - 【高亮/激活标签】：一个带有明显背景色和圆角边框的胶囊按钮，内部包含分类名称文本和数量指示徽章。
  - 【常规/未激活标签】：紧随其右侧，样式较弱（无背景填充），同样包含分类名称文本和数量指示徽章。

三、 任务详情卡片区块 (Task Item Card)
- 【任务标题】：位于卡片顶部的一行大字号加粗文本。
- 【状态标签组】(Tag Group)：水平弹性布局，包含若干个胶囊样式的小标签（通常用于指示任务的具体状态、类型或等级）。
- 【高亮信息块】(Highlight Block / Quote)：一个带有独立深色背景的矩形区块，左侧带有竖向的高亮边框线，内部是一段简短的当前进度或提示文本。
- 【任务描述段落】：一段占据多行的长文本，用于详细阐述任务的背景和要求说明。
- 【详情属性列表】(Details Grid)：位于卡片底部，采用“左侧标签-右侧内容”的两列网格布局（或Flex布局）。
  - 第一行：左侧为属性标签（如任务目标），右侧为具体的描述文本。
  - 第二行：左侧为属性标签（如任务奖励），右侧为具体的描述文本。

# 单个任务
整体界面是一个独立的【模态弹窗容器】(Modal / Dialog Container)，整体采用垂直布局。界面被清晰地分为头部的标题区和下方的详情列表区。

一、 头部区块 (Header Section)
- 【顶部弹性容器】：采用水平两端对齐的布局 (Flexbox, space-between)。
  - 【左侧信息容器】：垂直布局。
    - 上方为一行大字号加粗的【主标题文本】（如任务名称）。
    - 下方为一个【标签组容器】(Tag Group)，内部水平排列着两个带有圆角边框的胶囊样式【状态标签】。
  - 【右侧操作区】：一个包含“X”图标的圆形【关闭按钮】。
- 【实线分割线】(Solid Divider)：位于头部下方，将标题区与下方内容区隔开。

二、 详情内容区块 (Content Body Section)
该区域为垂直排布的列表，内部由6个结构相同的【信息展示块】(Information Block) 组成。相邻的展示块之间，均由一条【虚线水平分割线】(Dashed Divider) 隔开。

- 单个【信息展示块】的通用结构：
  - 【标题标签】：位于上方，是一行较小的辅助性说明文本（用于指示当前区块的属性名）。
  - 【详情内容】：位于标签下方，是一段常规大小的文本内容。根据属性不同，呈现为单行短文本或多行长段落。

- 具体列表结构（自上而下）：
  1. 【信息展示块 1】：属性名 + 短文本。
  2. 【虚线分割线】
  3. 【信息展示块 2】：属性名 + 短文本。
  4. 【虚线分割线】
  5. 【信息展示块 3】：属性名 + 短段落文本。
  6. 【虚线分割线】
  7. 【信息展示块 4】：属性名 + 多行长段落文本（详细描述）。
  8. 【虚线分割线】
  9. 【信息展示块 5】：属性名 + 单行文本。
  10. 【虚线分割线】
  11. 【信息展示块 6】：属性名 + 单行文本。
